GIT

What is Git?

A simple open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.

  • Small and fast
  • Distributed
  • Staging area
  • Branching and merging

Small and fast

Increases speed from 4x to over 300x

Distributed

Instead of doing a "checkout" of the current tip of the source code, the user does a "clone" of the entire repository, which means that every user has the exact same information and control over the full history of the repository.

Even when a centralized workflow is being used, every user of the repository has a full backup of the main server.

Staging area

The existence of the staging area allows you to decide which modified files are going to be part of the next commit.

There's no reason for commit everything that has changed, only those files that are pertinent.
